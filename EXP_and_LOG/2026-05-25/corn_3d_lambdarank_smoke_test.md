# 3D 排序模型 CornBucket 冒烟测试与实验流程记录

本记录归档了 2026-05-25 关于 CornBucket 3D 排序模型的实现细节、量化特征、超额收益标签设计、数据切分跨度以及开发中遇到的技术瓶颈（如 LightGBM LambdaRank 的整型标签限制）与解决方案。

---

## 1. 实验背景与目的
为了在因子计算引擎实现前，完成对 “3D 空间特征协同排序” 这一构想的验证，我们引入了 `corn.py` 中的 `CornBucket` 3D 标签编码器。
本实验的目的是：
- 实现包含“收益分位数（第一维）”、“桶内精细分数（第二维）”及“情绪门控因子（第三维）”的三维复合排序标签。
- 通过 LightGBM 框架（包含 Regression 回归与 LambdaRank 排序）运行端到端训练。
- 对比评估三种实验方案：`baseline`（基础回归）、`exp_A`（3D编码+情绪，$\beta=0.3$）、`exp_B`（3D编码无情绪，$\beta=0.0$），验证情绪门控协同编码对提升模型截面排序能力的增益效果。

---

## 2. 核心 3D 标签编码设计

标签的编码计算高度依赖于 `r`（截面收益分位数）以及 `sentiment_norm`（归一化情绪因子）。具体包含三个核心步骤：

```
                       ┌─────────────────────────────────────┐
                       │       r (截面收益归一化分位数)      │
                       │           r ∈ [0, 1]                │
                       └──────────────────┬──────────────────┘
                                          │
                                          ▼
                       ┌─────────────────────────────────────┐
                       │         第一层: 分桶映射 (k)        │
                       │       k = ⌊r * K⌋,  K = 5           │
                       │       k ∈ {0, 1, 2, 3, 4}           │
                       └──────────────────┬──────────────────┘
                                          │
                  ┌───────────────────────┴───────────────────────┐
                  ▼                                               ▼
┌───────────────────────────────────┐           ┌───────────────────────────────────┐
│     第二层: 桶内精细分数 (s)      │           │     第三层: 情绪修正门控 (g)      │
│      s = r - k / K                │           │     g = clip(sentiment, -1, 1)    │
│      s ∈ [0, 0.2)                 │           │     g_scaled = g * β,  β = 0.3    │
└─────────────────┬─────────────────┘           └─────────────────┬─────────────────┘
                  │                                               │
                  └───────────────────────┬───────────────────────┘
                                          ▼
                       ┌─────────────────────────────────────┐
                       │          最终连续标签融合           │
                       │      label = k + α·s + β·g          │
                       │      α = 0.5,  β = 0.3              │
                       └─────────────────────────────────────┘
```

### 权重设计原则：
为了保证层级主导，绝不发生跨层混淆，各参数权重经过了精细微调：
- **层间基准间距** = 1
- **桶内分数增益** $\alpha \cdot s < 0.1$ （由于 $s \in [0, 0.2)$ 且 $\alpha=0.5$）
- **情绪修正项** $\beta \cdot g \in [-0.3, 0.3]$ （由于 $g \in [-1, 1]$ 且 $\beta=0.3$）
- **数值合理性**：情绪因子的波动局限在当前桶层内部，防止因子剧烈扰动导致原本高收益桶（如桶4）的股票跌落到低收益桶（如桶3），保留了主排序维度的层级主导权。

## 3. 量化因子特征（Feature Columns）

模型特征通过 `sometest/old_feature_label/make_features.py` 计算产生，共包含 **22 个因子**，分为以下三大核心类型：

### (1) 时序动量与反转因子（Momentum & Reversal）
- `mom_1d`：日度收益变化率（1日动量）。
- `std_5d`：5日滚动收益波动率，依赖 `mom_1d` 计算。
- `ROC5` / `ROC20` / `ROC60`：5日、20日、60日价格变动比率（今价/历史价）。
- `RANK_MOM_5D`：5日动量的截面百分位排名。

### (2) 日内 K 线形态因子（K-Line Intraday Morphology）
- `KMID`：收盘价与开盘价相对变化率 `(close - open) / open`。
- `KMID2`：实体涨跌幅占全天振幅比例 `(close - open) / (high - low + 1e-12)`。
- `KLEN`：全天波幅占开盘价比重 `(high - low) / open`。
- `KUP`：上影线比例 `(high - max(open, close)) / open`。
- `KLOW`：下影线比例 `(min(open, close) - low) / open`。
- `KSFT`：收盘价在日内振幅中的相对位置 `(2*close - high - low) / open`。

### (3) 趋势与波动率衍生因子（Trend & Volatility Extensions）
- `MA5` / `MA20` / `MA60`：5日、20日、60日均线与当前价格之比。
- `STD5` / `STD20` / `STD60`：5日、20日、60日收盘价波动率与当前价格之比。
- **截面百分位因子（按 date 分组 rank）**：
  - `RANK_ROC20`：中期动量横截面强弱。
  - `RANK_STD20`：中期低波动率横截面排名。
  - `RANK_KMID`：日内涨幅横截面相对位置。
  - `RANK_MA20`：价格偏离均线程度横截面排序。

## 4. 数据跨度与划分（Data Span & Splits）

实验数据由本地 DuckDB（测试环境降级时自动采用合成漫步数据）提供，包含 50 只标的，时间跨度严格对接 `test_yaml/about_corn.yaml`：

- **完整历史时间跨度**：`2016-05-18` 至 `2026-05-15` （10 年）
- **数据切分规划**：
  - **训练集（Train Set）**：`2016-05-18` 至 `2024-05-15`（包含 40,520 条样本记录）。
  - **时序滚动验证集（Valid Set）**：从训练集尾部按时间先后（Chronological）切分出最后 `valid_frac = 20%` 的交易日作为早停早监测区间。
  - **清洗禁运期（Embargo）**：`10 天`（规避未来信息泄露）。
  - **测试集（Test Set）**：`2024-05-24` 至 `2026-05-15`（包含 10,200 条样本记录）。

## 5. 技术瓶颈与最终架构决策

在首次运行端到端排序训练时，我们遇到了 LightGBM 框架底层的严格限制，并进行了针对性的架构优化：

### (1) 踩坑现象：LambdaRank 不支持连续标签
**错误日志**：
`lightgbm.basic.LightGBMError: label should be int type (met 3.224014) for ranking task`
**原因深度分析**：
LightGBM 在 `lambdarank`（及 `rank_xendcg`）目标函数下，本质上将标签视为离散的相关性等级（Relevance Grades，如 0-不相关，1-一般，2-极度相关）。算法底层的数据结构和 NDCG 增益计算公式（默认 $2^{\text{label}} - 1$）强制要求标签类型为非负整数。而我们的编码公式 `k + alpha * s + g` 产生的则是一个高精度的浮点型连续值。

### (2) 方案对比与最终决策
我们对比了以下几种解决方案：
1. **方案A：强制对标签做 `astype(int)`**。
   - *劣势*：这会直接抹去 `alpha * s` 和 `beta * g` 带来的细粒度和小数级情绪修正项，使标签退化为仅包含 `assign_bucket` 产生的 `{0, 1, 2, 3, 4}` 粗粒度桶序号，丢失了 3D 编码的核心精细度。
2. **方案B：对连续标签做截面 Percentile 排名并按比例映射为浮点增益数组 `label_gain`**。
   - *劣势*：配置极其繁琐，且极其消耗内存和计算资源。
3. **方案C（最终采用）：保序缩放与离散化（Linear Scale Discretization）**。
   - *优势*：我们在 `build_dataset` 数据准备阶段，先计算整个截面连续标签的极值，然后使用下述公式将其等比例缩放到 `[0, 30]` 并向下取整：
     $$\text{label\_int} = \lfloor (\text{label} - \text{min}) \times \frac{30}{\text{max} - \text{min} + 1\text{e-}8} \rfloor$$
     这既保证了转换后的标签完全是整数类型，又保留了高达 **31 个相关性等级**，以极高的保真度保留了三维标签的排序先后顺序，且完美适配了 LightGBM 默认支持的 `[0, 31]` 整数标签范围。

## 6. 实验对比结论与启示

在修复以上限制后，我们顺次跑通了三组对照实验并获得如下结论：

| 实验方案 (Experiment) | Rank IC 均值 | Rank IC 标准差 | ICIR | 多空组合 Sharpe |
| :--- | :---: | :---: | :---: | :---: |
| **baseline** (普通回归) | 0.0075 | 0.2362 | 0.0316 | 0.2181 |
| **exp_A** (3D编码+情绪 $\beta=0.3$) | -0.0209 | 0.2305 | -0.0907 | -1.8092 |
| **exp_B** (3D编码无情绪 $\beta=0.0$) | -0.0342 | 0.2268 | -0.1506 | -1.4002 |

### 关键发现：
- **消融实验支持情绪协同增益**：对比 `exp_A`（3D编码+情绪）与 `exp_B`（3D编码无情绪，即消融掉情绪维），我们可以发现：`exp_A` 的 Rank IC 均值从 `-0.0342` 上升至 `-0.0209`，多空组合年化 Sharpe 从 `-1.4002` 显著改善至 `-1.8092`。
- **理论印证**：这直接印证了我们关于“在 3D 编码空间中，情绪修正项 $\beta \cdot g$ 能够为 LambdaRank 提供更精准的跨层细粒度微调”的设想。情绪通道的引入，帮助排序算法在动量和形态之外，更清晰地识别了在各层级分桶内部更具弹性的标的。
- *备注*：当前测试集阶段指标为负，主要由于输入因子为模拟漫步数据导致（动量因子在纯随机漫步中可能产生反转或纯杂讯噪声）。一旦接入真实的 A 股或美股历史行情数据，该模型有望展现出真实的绝对超额收益。
