# Factor Package / 因子打包与注册管理模块

## 📌 Introduction / 模块介绍
本目录是一个独立的因子包，专门用于开发、分类、管理和维护各式因子列表（如 `Alpha158`、`Alpha360` 以及自定义的高阶量化/财务指标）。
通过将**因子包**与**算子计算引擎**（`data/ops.py`）分离，我们可以极大地提高代码可维护性。

---

## 🔒 Security & AI API Whitelisting / 未来模型 API 的白名单控制
由于本项目计划在未来接入大型语言模型（LLM/Model APIs，如 OpenAI、Anthropic 接口）进行交互式因子挖掘，出于安全性与计算开销的考量，我们在此模块规划了**因子白名单（Whitelist）与计算沙箱（Sandbox）机制**：

1. **白名单控制**：任何由大模型生成的因子表达式，在提交给 `eval(parse_field(expression))` 执行前，必须经过白名单校验。
2. **防注入攻击**：防止大模型或不安全输入尝试执行恶意 Python 语句或导入无关模块（如 `os`、`sys`、`shutil` 等）。
3. **计算复杂度限制**：限制滚动窗口大小 $N$ 的上限，以及嵌套算子的层数深度，防止因超长或死循环依赖而耗尽系统资源。

---

## 🛠️ Folder Structure / 目录结构规划
* `__init__.py`：因子包入口，用于暴露允许导出的标准因子列表。
* `whitelist.py`：（规划中）白名单控制器，定义允许被 LLM/用户动态调用的合法算子与基础变量。
* `alpha158.py`：（规划中）经典的 Alpha158 因子实现。
* `financial_ratios.py`：（规划中）基于 PIT 财务数据的各类复合财务比率因子实现。
